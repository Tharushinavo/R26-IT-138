# MathsMates — Personalized Maths Learning Engine

An AI-powered adaptive learning engine that personalizes maths activities for children with dyscalculia by analyzing student interaction data during learning sessions. The system dynamically tracks learner performance and adjusts activities using Bayesian Knowledge Tracing (BKT) to improve mathematical understanding and skill mastery.

• Personalized Learning Pathways
• Skill Mastery Estimation
• Adaptive Difficulty Adjustment
• Misconception Detection
• Real-time Feedback Generation

The system is specially designed to support children with dyscalculia by adapting learning experiences according to student responses, response time, attempts, and error patterns.

Component 1 of 4 in the full MathsMate adaptive learning ecosystem.
The cognitive profile generated by this component is used by:
•	Personalized Learning Engine 
•	Adaptive Game Engine 
•	Learning Insight & Early Warning System 
to provide adaptive learning support and intervention recommendations.

**Data flow:**

1. Student interacts with a math activity through the Expo React Native mobile app.

2. The app captures interaction data such as:
   - Answer submission
   - Response time
   - Number of attempts
   - Error patterns

3. The interaction data is sent to the FastAPI backend through REST API requests.

4. The FastAPI server validates and processes the received data.

5. The Bayesian Knowledge Tracing (BKT) engine updates the learner’s mastery level based on student performance.

6. The Adaptive Learning Engine determines:
   - Next suitable activity
   - Difficulty level
   - Personalized learning pathway

7. If the answer is incorrect, the Hint Generation Module creates context-aware feedback.

8. Processed learner data and interaction logs are stored in Supabase PostgreSQL database.

9. The backend generates a personalized response containing:
   - Updated mastery status
   - Adaptive feedback
   - Next learning activity
   - Generated hints

10. The response is returned to the Expo mobile application.

11. The mobile UI updates dynamically and displays:
   - Feedback
   - Hints
   - Next adaptive activity
   - Learning progress

## Quick start

### 1. Backend (Server)
powershell
cd Personalized_Maths_Learning_Engine


# Activate virtual environment
.\myenv\Scripts\activate

# Install dependencies
pip install fastapi uvicorn pandas scikit-learn supabase

# Start FastAPI server
uvicorn main:app --reload --host 0.0.0.0



Server runs on **port 8000**.
Swagger UI → http://localhost:8000/docs



### 2. Frontend (Client)

In a **new, separate terminal**:

npx expo start -c

Scan the QR code with **Expo Go** on your phone.


### 3. Database (Supabase)

Create a Supabase project.

Configure the Supabase connection in:
lib/supabase.ts

EXPO_PUBLIC_SUPABASE_URL=https://your-project.supabase.co


### 4. ML Model

The Personalized Math Learning Engine uses Bayesian Knowledge Tracing (BKT) to dynamically estimate student skill mastery and adapt learning activities based on learner performance.

If mastery level is high → the system provides more advanced learning activities.
If mastery level is low → the system provides simpler questions, additional practice, and personalized feedback.

The engine continuously analyzes:

Student answers
Response time
Number of attempts
Error patterns

to generate adaptive learning pathways and real-time feedback for children with dyscalculia.



For production-quality training, use Google Colab with a larger synthetic dataset (3000+ rows).
